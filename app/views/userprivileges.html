Properly manages user privileges constitute in a big extent for the security of the industrial control systems. A user privilege is a right to execute a particular type of command, or a right to access another user's object within the system. The types of privileges are defined by the administrator of the system. If users are provided with unnecessary system privileges or data access rights, then the system has an increased vulnerability for potential deliberate or accidental attacks.

To battle this issue companies have to undertake a series of actions which should result in all users of ICT systems being only provided with the privileges that they need to do their job. This principle is often referred to as ‘Least Privilege’.

There are many risks originating in inappropriate user privileges of ICSs. Some notable examples applicable to ICS are the misuse of privileges, increased attacker capability or negating established security controls.

Privileged users are attackers’ primary target as they have access to various parts of the system and rights associated with the system, thus they number should be kept as low as possible. Moreover as pointed by ENISA the privilege abuse constututed for 70% of all cases of insider attacks in 2016

Risks (according to NCSC)
  • Misuse of privileges - Privileged users can make make unauthorised changes to the configuration of systems, leading to a loss or leakage of the of information from ICS
  • Increased attacker capability - Attackers can use compromised user accounts to conduct the attacks and they also can return and reuse or sell the this account in the future. The system privileges provided to the compromised account will be available to the attacker
  • Negating established security controls - As soon as the attackers wil gain the privileged access to ICT systems they will attempt to make changes to security controls and cover their tracks by deleting audit logs so that their activities are not detectable
Recomendations
The top 5 things organisations should do to effectively manage user privileges:
  1. Decide what privileges user needs - system users need to undergo some form of screening to a level that is commensurate with the sensitivity of the information they will have access to, it can allow the employer to properly set the account privileges within the system
  2. Managage and review user privileges - strict procedures should manage accounts, it should comply with the ‘Least privilege’ principle. Highly privileged accounts should be reviewed more often than the accounts of normal users
  3. Introduce standards for privileged users identification and access control - system itself should enforce complicated, alphanumeric passwords which can limit the possibility of gaining the access to privileged account by the intruder. Moreover the log-in highly-privileged accounts should involve two-factor authorisation
  4. Limit the number and use of privileged accounts - strictly control the number of privileged accounts for roles such as administrators. Make sure that those accounts are not used for high risk or day to day activities, for example to gain access to external email or browse the Internet
  5. Monitor all users - monitor user activity, particularly all access to sensitive information and the use of privileged account actions, such as the creation of new user accounts, changes to user passwords or the deletion of accounts and audit logs
